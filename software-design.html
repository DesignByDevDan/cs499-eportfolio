Artifact Overview

This artifact represents the Software Design and Engineering enhancement area for my CS 499 ePortfolio. The underlying project is the C-Cure Reporting Database, a SQL Server solution designed to support compliance-driven access eligibility reporting.

I selected this artifact because it demonstrates real-world system design concerns: maintainability, readability, separation of responsibilities, repeatable processing, and clear communication of intent through structure and documentation.

Original Functionality

Before enhancement, the solution supported eligibility evaluation and delta-style reporting outputs. However, logic was tightly coupled, harder to follow, and more difficult to extend safely.

Enhancement Implemented

The enhancement focused on refactoring the stored procedure to improve modularity, separation of concerns, naming clarity, and maintainability without altering intended functional behavior.

Key Improvements

  • Clearer separation of data preparation and evaluation logic
  • Reduced duplicated logic patterns
  • Improved naming consistency
  • Added intent-focused documentation comments
  • Improved structural readability for safer future modifications

Course Outcome Alignment

  1. Collaborative Environments: Clearer code structure improves peer review and shared maintenance.
  2. Professional Communication: Structured documentation and readable organization support stakeholder understanding.
  3. Algorithmic Trade-offs: Refactoring balances clarity and performance while reducing future defect risk.
  4. Tools and Techniques: Demonstrates professional SQL Server development and refactoring discipline.
  5. Security Mindset: Improved structure reduces risk of unintended side effects or logic flaws.

Reflection

This enhancement reinforced that high-quality software engineering is not only about producing correct results, but about creating solutions that are understandable, extensible, and safe to evolve. The refactoring process required careful validation to ensure behavior remained consistent while improving structural clarity.

This foundation directly supports the algorithmic and database enhancements that follow.
