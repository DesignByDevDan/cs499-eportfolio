Artifact Overview

This artifact represents the algorithms and data structures component of my CS 499 ePortfolio. It builds upon the system reviewed in Milestone One and focuses on improving the core logic used to evaluate access eligibility and generate “delta” outputs. The enhancement emphasizes correctness, efficiency, and clarity of the decision-making process.

Problem Being Solved

The underlying problem is to recompute access eligibility changes (deltas) based on updated inputs, then determine which changes must be applied downstream. The process must reliably identify additions, removals, and updates while handling real-world data conditions such as incomplete records, timing differences, and repeated runs. The algorithm must be deterministic and maintain consistent results.

Algorithmic Approach and Improvements

The enhancement refines the evaluation and delta-generation logic so that the system minimizes unnecessary recomputation and avoids redundant outputs. The enhanced approach prioritizes set-based processing, reduces repeated scanning of the same data, and enforces a clear sequence of operations: (1) normalize inputs, (2) compute candidate eligibility states, (3) compare current vs prior states, and (4) emit only actionable changes. This results in a more efficient and maintainable algorithm.

Data Structures and Data Handling

Although the implementation is written in SQL, the enhancement still demonstrates data structure thinking through consistent use of keyed datasets (e.g., person identifiers, policy/group identifiers, and effective date windows). The improved logic treats intermediate results as structured sets that can be joined and compared reliably, which reduces complexity and improves correctness of delta detection.
